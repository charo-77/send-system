## 3. 已验证成功的关键实现

### 3.1 导入文档
稳定做法：
- 清新手引导/遮罩
- 对 `插入` 发送完整 pointer/mouse 事件链
- 在菜单中按以下优先级定位：
  1. `.l-icon-BjhBasicDaoruwendang`
  2. 文案 `导入文档`
- 上传器优先匹配：
  - `input[type=file][name=file][accept*=".docx"]`

### 3.2 封面分流策略
#### 无图文章
稳定做法：
- 进入 `AI封图`
- 点击 `根据全文智能生成封面`
- 慢轮询等待确认可点
- 在 **AI封图页签内部** 完成确认
- 不过早切 tab / 不过早关弹层

#### 单图文章
稳定做法：
- 切单图
- 通过封面组件 React props 的 `open()` 打开选择器
- 选择导入图片
- 点击 `确定 (n)`

#### 三图文章
稳定做法：
- 切三图
- 同样通过 React `open()` 打开选择器
- 自动补足三图选择
- 点击 `确定 (n)`

### 3.3 活动选择
稳定做法：
- 封面完成后统一执行活动逻辑
- 有指定活动名：按目标选
- 无指定活动名：自动选择第一个可见活动
- 发布前页面需看到：
  - `已添加`
  - `活动详情`

### 3.4 发布成功判定
主线现在用这些标志判断发布成功：
- `提交成功，正在审核中`
- `查看发布状态`
- URL 中含 `/builder/rc/clue`

## 4. 不要再重复踩的坑

1. **不要再用固定菜单索引点导入文档**
   - 会误点到“表格”
2. **不要再把无图 AI 封图切回本地上传再选图**
   - 这条路已经证明不稳定
3. **不要过早关闭 AI 封图弹层**
   - 先等官方流程自己收口
4. **不要只在传 `--activity` 时才执行活动选择**
   - 正式主线必须默认执行活动逻辑
5. **不要在人工验收时让窗口成功后自动关闭**
   - 用 `--keep-open-after-success`
6. **不要在人工观察阶段双开窗口**
   - 容易后台化/抢焦点，看起来像“没开”
